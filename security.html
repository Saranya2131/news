PYTHON SECURITY TIPS

1. Input Validation and Sanitization:
Always validate and sanitize all external data, including user inputs, API responses, and file uploads, to prevent injection attacks (e.g., SQL injection, XSS). Avoid using eval(), exec(), or os.system() with untrusted input due to inherent security risks.

2. Dependency Management:
Keep Python and all installed libraries updated to the latest stable versions to benefit from security patches. Use tools like pip-audit or safety to scan for known vulnerabilities in your project's dependencies. Minimize the number of dependencies to reduce the potential attack surface.

3. Secure Handling of Secrets:
Never hardcode sensitive information like API keys, database credentials, or cryptographic keys directly in your code. Utilize environment variables or dedicated secret management tools for storing and retrieving secrets securely.

4. Authentication and Authorization:
Implement strong authentication mechanisms and robust authorization controls to ensure only authorized users and processes can access sensitive resources. Use secure libraries for password hashing (e.g., bcrypt, argon2-cffi) and secure session management (e.g., Flask-Login).

5. Secure File Operations:
When handling file uploads, validate file types (using MIME types, not just extensions), sanitize filenames, and store uploads outside the application's root directory. Implement size limits for uploaded files to prevent Denial-of-Service (DoS) attacks.

6. Error Handling and Logging:
Implement proper error handling to avoid revealing sensitive information in error messages to users. Log detailed errors internally for debugging. Be mindful of what information is logged to prevent inadvertently exposing secrets.

7. Secure Transmission and Encryption:
Use secure protocols like HTTPS for data transmission over networks.Encrypt sensitive data at rest using libraries like cryptography when storing it.

8. Code Security Analysis:
Regularly perform static analysis of your code using tools like Bandit to identify potential security vulnerabilities. Conduct code reviews to catch security flaws early in the development cycle.

9. Least Privilege Principle:
Grant the minimum necessary permissions to users and processes to perform their intended functions, limiting potential damage in case of a breach.

10. Virtual Environments:
Use virtual environments (e.g., venv, pipenv) to isolate project dependencies and prevent conflicts or security issues from affecting other projects or the system's global Python installation.
